# Drawing-tools roadmap — architecture-convergence order (2026-06-29)

## Context

The tiling ENGINE is done and proven (7 tilings, GPU dab pipeline, persistent live tiling, configurable tile/brush, `EditorCore` `@Observable` single source of truth, headless render harness). The next phase adds drawing TOOLS. The explicit goal (user): build these tools NOT primarily to ship features, but to **stress the architecture with diverse tool shapes so the core converges** — installing each reusable seam against the cheapest possible client — BEFORE the big investment in a **high-performance raster brush system** (the "make-or-break" feature). Secondary: shippable value along the way; minimal rework.

Decided via a 3-lens design workflow (architecture / brush-endgame / pragmatist) grounded in a fresh read of the data model, commit path, blend/pipeline, transform plumbing, and coord spaces. All three lenses converged independently on the same order and the same pivotal ruling.

## Current architecture (the seams these tools must install)

*   **Data model:** emit-and-forget. `StrokeSample` → `StrokeSession.emit` (interpolate in world, fold to canonical, wrap-placements) → `[EmittedDab]` / `DabSpec` → GPU `Dab`/`DabInstance` (32-byte stride). After commit, **only pixels survive** — no stroke/dab list retained.
*   **Textures:** one **flat canonical** (source of truth, `loadAction=.load`) + **one live tile** (in-progress stroke). `commitOverlay()` rasterizes live dabs into canonical, clears the array. `tiling_fragment` composites `live over committed`. No layers.
*   **Pipeline:** one fixed `commitDabPipeline`, blend baked in; `dab_fragment` outputs hardcoded black `float4(0,0,0,a)` (only alpha varies, no color plumbed).
*   **Transforms:** none. `WrappedPlacement` carries a translation only; mirror/p2 bake the isometry by re-centering the ROUND dab (radial symmetry ⇒ no 2x2 matrix). No affine anywhere.
*   **Tool state:** none. `EditorModel` has brush/tiling/tileSize/showGrid config only; no `activeTool`.
*   **Pressure is QUANTIZED PER SEGMENT (pre-existing limitation, not color-related).** `StrokeSession.emit` computes ONE radius + ONE alpha from the incoming sample's pressure and stamps the WHOLE interpolated segment (`p_prev`→`p_cur`) with it (the `StrokeInterpolator` walks POSITIONS only; it carries no pressure). So radius/alpha step at each input sample rather than tapering smoothly along a segment. **Deferred follow-up** (own step, brush-era): per-dab interpolated pressure — interpolator emits `(position, pressure)` and `emit` computes radius/alpha/color PER DAB from arc-length-lerped pressure. Colored brush (step 1) intentionally does NOT fix this — color rides as a constant-per-segment attribute exactly like radius/alpha, inheriting (not worsening) the quantization.

## The decided order

1.  ~~Colored brush + picker~~ **DONE (2026-07-01)** — per-dab-attribute seam installed: straight RGBA `color` on `DabSpec` → `StrokeSession` → `Dab`/`DabInstance` (stride 32→48, `LayoutTests` -guarded), shader outputs picked RGB with alpha = pickedAlpha × pressureAlpha, native SwiftUI `ColorPicker` bound to `EditorModel.inkColor`. The eraser (2) and raster brush (5) reuse this per-dab attribute channel + the stride-48 layout.
2.  ~~Eraser~~ **DONE (2026-07-02)** — `activeTool` (draw/erase, first tool-mode) added to `EditorModel`; per-dab `blendMode` reused the `_pad0` slot (stride still 48, `LayoutTests` -guarded); batched by mode at commit via a destination-out pipeline (`dst *= 1-src.a`); live preview reused the existing tiling composite (erase = a `(0,0,0,coverage)` dab); toolbar button + `e` key. Headless-proven by the `eraser` harness scene (bounded erase: center alpha→0, surrounding ink survives). Undo (step 3) wraps the same `commitOverlay` seam; the raster brush (step 5) reuses the per-dab blend vocabulary.
3.  ~~Undo / redo (snapshot)~~ **DONE (2026-07-02)** — snapshot-based history at the `commitOverlay()`/`clearCanvas()` seam: a new `UndoHistory` (in `MetalRenderer`) holds a bounded ring (20-deep) of full-tile canonical `MTLTexture` snapshots, captured SYNCHRONOUSLY before each commit/clear and blit back on undo/redo (redo captures current state first → symmetric). Undo/redo/clear are edge-triggered `CommandToken`s on `EditorModel` flowing model→view via the coordinator's `applyXIfChanged`; enable-state flows view→ContentView via an `onHistoryChanged` callback (the `onZoomChange`/`onStrokePoint` idiom). ⌘Z/⇧⌘Z via a modifier-aware `EditorKeymap.handle(key:command:shift:_:)` overload + toolbar arrows. Tile-size/tiling changes flush history (snapshots are dimension/fold-specific). Chose **full-tile snapshots over the originally-planned dirty-rect deltas** — simpler + geometry-agnostic (covers draw, erase, clear uniformly); optimize to deltas only if memory bites. Headless-proven by the `undo-redo` scene (commit A left + B right → undo → A survives, B blank; negative-control verified). Selection (step 4) reuses this commit-transaction seam → its composite is undoable for free.

## Current position (2026-07-13)

Steps 1–5 are DONE and on `main`. After the raster brush the work was the **professional-brush-engine pass** (step 5.5, plan `15`) plus an infrastructure hardening pass — the `EditorTransaction` lifecycle module (`14`) that ended a recurring class of edit-transaction-teardown bugs (four review rounds, merged `4a39787`).

**Layers (step 6) is the next feature.** It is a pure consumer of seams already installed — per-dab attributes (step 1), snapshot undo at the commit seam (step 3), the shared provisional channel + affine composite (step 4), and the hardened edit-transaction lifecycle (`14`). It multiplies canvas state but introduces no new engine mechanism.

## Per step — seam installed, change, what later steps reuse

1.  Colored brush — seam: per-dab attributes. Add `color` to `BrushDynamics` → `DabSpec` → `Dab`/`DabInstance` (stride 32→48, extend the layout-parity test — same grow-and-guard pattern the clip fields used), un-hardcode `dab_fragment`, add `color` + paper color to `EditorModel` + a picker. Cheapest tool, most-reused seam: eraser's blend field, the brush's hardness/brush-id all slot into the attribute mechanism this establishes. Harness check: render a colored stroke, pixel-assert the dab center is the picked color (not black).
2.  Eraser — seam: tool mode + blend-as-per-dab-state. Add `activeTool` to `EditorModel` (first tool-mode concept; `draw` default). Add a per-dab `blendMode` (the attribute slot from step 1), **batched by mode at `commitOverlay()`** — prefer this over a one-off destination-out pipeline, because per-dab/batched blend generalizes to multiply/screen and the textured brush; a destination-out-only pipeline does not. Breaks the single-fixed-pipeline assumption against a trivial client. Harness check: stamp, erase over it, pixel-assert canonical alpha → ~0 in the erased region.
3.  Undo/redo — seam: commit boundary as geometry-agnostic transaction. DONE (2026-07-02). SNAPSHOT-based (see ruling below), at the `commitOverlay()`/`clearCanvas()` seam the live-tile note already names. Shipped as a bounded ring (20-deep) of **full-tile canonical snapshots** (`UndoHistory` in `MetalRenderer`, `MTLBlitCommandEncoder` copies) — NOT the dirty-rect deltas originally sketched here; full-tile is simpler and equally geometry-agnostic, and the delta optimization is deferred until memory actually bites. Snapshot captured synchronously BEFORE each commit/clear; undo = blit back, move to redo (redo snapshots current state first → symmetric); new commit clears redo. Done at step 3 (NOT last) so it wraps commit BEFORE selection introduces a non-stroke canvas mutation → selection becomes undoable for free. Proven headlessly exactly as predicted: commit A → commit B → undo → assert canonical == post-A (the `undo-redo` harness scene, negative-control verified).
4.  Selection + transform — seam: affine 2x2 + canonical↔world inverse + provisional channel. DONE (2026-07-03). The deepest tool, and the one deliberate forward bet — shipped as designed. Installed: the affine textured-quad through a vertex matrix (`selection_vertex`/`selection_fragment` + the `Affine` `float3x3` value type; the fast round-dab path for plain draw was kept untouched, NOT retired — the selection path is a separate pipeline); `canonicalToWorld` promoted to a real API on all 5 strategies (point round-trip proven for all; cell round-trip for grid/mirror/rotational + half-drop/brick phase-safe half — half-drop/brick return the raw-floor cell on the phased axis, an intended property); pixel extraction via blit (`CanonicalRaster.extractRegion`) + a `clearRegion` cut (staging-blit overwrite) + a transform+composite pass (`SpikeRenderer.compositeSelectionIntoCanonical`). Reuses `activeTool` (2) and snapshot undo (3) — a confirmed transform is one undo entry via the `transformRestore` stash pushed onto the ring. The affine IS **the same matrix the textured/rotated brush stamp needs** — build-once-brush-inherits realized. **Draft mode shipped** via the shared PROVISIONAL channel (3rd tiling-shader texture slot, `provisional over live over canonical`, bit `0x40000`): the transform re-renders the lifted selection into it every gesture (canonical untouched), confirm bakes, cancel discards. The interactive scale/rotate HANDLES are the deferred manual layer (move-drag + the scriptable-matrix seam shipped and are GPU-proven); warp/lasso/feather/cross-fold deferred.
5.  Raster brush system (make-or-break). Inherits attributes (1), blend vocabulary (2), commit-history (3), affine matrix path (4) — with emit-and-forget intact (what a low-latency emit→stamp→commit loop wants). Adds textured stamps, hardness, spacing, smoothing on a struct that already grew, a pipeline that already blends per-dab, a history that already wraps commit, a vertex matrix that already exists. No seam invented under latency pressure. (Correctness is harness-verifiable; **latency is not** — needs on-device timing.)
    5.5. Professional brush engine pass (before layers). Deepen the v1 brush into product-grade modules. `BrushInput` grows beyond position+pressure; `BrushDynamics` becomes the single evaluator for size/flow/opacity/spacing/scatter/rotation/color/wetness; `LiveStroke` may retain transient samples/dabs during an active stroke for taper, prediction correction, and wet behavior while keeping emit-and-forget for committed document storage; `BrushRenderer` gains material modes; `BrushStudio` replaces the clunky settings popover with library + live preview + grouped controls. This is the new product gate before layers because layers would multiply every brush-quality problem across more state.
6.  Layers (deferred past the professional brush pass). N canonical textures + composite pass (reuse selection's) + per-layer routing + layer stack + per-layer snapshot undo. Pure consumer — installs nothing the brush needs; step 2's per-dab blend already gives layers their compositing vocabulary, so deferring costs zero rework.

## Pivotal ruling: emit-and-forget stays; undo is snapshot-based

**Keep emit-and-forget through all 5 tools AND the brush. DO NOT build a retained/vector stroke model. The canonical TEXTURE is the retained artifact; undo operates on it, not on dabs.**

Undo with emit-and-forget works precisely because every canvas mutation funnels through one chokepoint — `CanonicalRaster.commit`. Undo wraps that: snapshot canonical before each commit, blit back to undo. It's **geometry-agnostic**, so it covers draw strokes, eraser strokes (no clean dab list), and selection transform-composites (no originating dabs at all) uniformly — which a dab-list model would have to special-case. And the brush will churn the dab schema, so a retained dab list built now would be re-versioned or made lossy; snapshots are schema-proof. (This supersedes the live-tile note's earlier "keep the dab list as source of truth" lean — that model is right only for re-fold-on-tiling, which is deferred.)

## Draft transform: confirm-to-bake via the ephemeral layer

The transform tool must let the user **play with rotate/scale/warp** and see a live draft, baking only on **confirm**. The substrate for this is the **ephemeral layer** documented in `2026-06-22-live-tile-perf-promotion-undo.md` (a second draft texture composited `ephemeral over live over canonical`, cleared+repainted every input event, never baked until promoted) — but that note framed it for predicted touches only. **Generalize it to a shared provisional render channel.** The transform flow:

*   **Enter transform:** blit the selected region's pixels out of canonical into a selection texture.
*   **Drag handles (rotate/scale/warp):** re-render the selection texture through the **affine path** (the seam selection installs) into the **ephemeral layer** every gesture event. Canonical is untouched → smooth, discardable, re-rendered per frame. This is exactly what the ephemeral layer is for.
*   **Confirm:** composite the final transformed result into canonical via the commit seam → undoable (step 3).
*   **Cancel:** clear the ephemeral layer; canonical was never touched.

So the ephemeral layer is built **with the selection tool (step 4)**, as a general channel, not a prediction-only stub. The brush's lookahead later reuses the same channel.

## Overrides vs the user's first-pass order

User proposed: eraser → selection → colored brush → layers → undo. Reordered to color → eraser → undo → selection → brush → (later) layers. Color first (cheapest installer of the most-reused seam); undo 3rd not last (must wrap commit before selection mutates the canvas non-stroke-shaped, and is the cleanest headless check); selection 4th not 2nd (deepest tool — must land on mature seams, and its affine work is the brush's); layers explicitly last (pure consumer, defer past make-or-break).

## Top rework risks this order avoids

1.  **Building retained strokes early** — deepest avoidable seam; abandons emit-and-forget for zero latency benefit and gets re-versioned by the brush. Avoided by the snapshot/blit ruling.
2.  **Selection too early** — most surface area; on immature seams every later change ripples through it. Avoided by placing it after tool-mode (2) + undo (3).

## Verification posture

Steps 1–4 are largely headless-verifiable via the render harness (color, erase-to-transparent, undo-restores-snapshot, transform-lands-where-the-matrix-predicts) — extend the harness scene catalog per step. Manual-only residue: selection's marquee/handle UI, and the brush's latency (on-device timing). Engine/`EditorCore` logic stays unit-tested under `swift test`.
